Version: 3.3

Date adapters#

As of version 3 of Charba, Chart.JS is not longer provided by the bundle artifact where there was a default date time library to use for time series.

The Chart.JS time scale requires both a date library and corresponding adapter to be present.

Chart.JS is providing the integration with 3 date libraries, that you can choose by the specific resource type:

  1. Moment.js which remains the default for Charba, enabled by EmbeddedResources or DeferredResources classes.
  2. Luxon which can be enabled by LuxonEmbeddedResources or LuxonDeferredResources classes.
  3. Date-fns which can be enabled by DatefnsEmbeddedResources or DatefnsDeferredResources classes.

Moment.js#

Moment.js is a lightweight JavaScript date library for parsing, validating, manipulating, and formatting dates.

These are the defaults that the library implements that you can change into cartesian time axis.

NameDefaultExample
millisecondh:mm:ss.SSS a7:25:34.639 pm
secondh:mm:ss a7:25:34 pm
minuteh:mm a7:25 pm
hourhA7PM
dayMMM DFeb 19

monthMMM YYYYFeb 2020
quarter[Q]Q - YYYYQ1 - 2020
yearYYYY2020

To see all available formats, have a look here into Moment.js documentation.

Luxon#

Luxon is a powerful, modern, and friendly wrapper for Javascript dates and times.

These are the defaults that the library implements that you can change into cartesian time axis.

NameDefaultExample

To see all available formats, have a look here into Luxon documentation.

Luxon is the only adapter which can be configured in order to update some behavior during parsing, formatting and date management.

The Luxon options has got the following table options:

NameTypeDefaultDescription
zoneStringnullIf defined, adapter will use that time zone. See here for more details
localeStringnullIf defined, adapter will use that locale. See here for more details
setZonebooleanfalseIf true, adapter will apply the zone as fixed-offset one
outputCalendarStringnullIf defined, adapter will use that calendaring systems. See here for more details
numberingSystemStringnullIf defined, adapter will use that numbering systems. See here for more details

Using the date adapter#

Charba enables the possibility to get a date adapter instance from Chart.JS which can provides some capabilities, like formatting, parsing and date operations.

Here is an example how to use a date adapter to format ticks by callback:

CartesianTimeAxis axis = new CartesianTimeAxis(chart);axis.getTicks().setCallback(new TimeTickCallback() {
    private DateAdapter adapter = new DateAdapter();
    @Override    public String onCallback(Axis axis, Date value, String label, int index, List<TimeTickItem> values) {        return adapter.format(value, "yyyy");    }});
